## ✨ 项目特色

### 🎭 不是AI写稿，是真人开会

NPC 发言禁止"首先其次总之"、禁止自我介绍式开场，要求口语化、允许打断、鼓励引用自身经历。每段发言强制包含**建设性提案 + 现实约束**——不是和稀泥的走过场，是真实编前会的压力测试。

> ❌ "作为一名资深策划，我认为这个选题有很大的潜力。首先……"
> ✅ "这个选题能打！年轻人对老字号的好奇心被低估了——我上次做《老字号盲测》的时候，评论区全在问'这居然是老牌子？'。方案是3分钟竖版，开场用反差，先不说是哪家，最后揭秘。"

### ⚡ 不回避冲突，制造有价值的碰撞

三类冲突机制覆盖真实编辑部的核心博弈：**创意vs成本**（策划提大制作，后期泼冷水）、**流量vs深度**（运营要标题党，把关人亮牌）、**互相激发**（记者抛素材，策划追加创意）。4级冲突强度 + 红黄绿牌审核机制——争论有建设性，不是为吵而吵。

### 🧬 七维面板，每个NPC都有灵魂

31位预设NPC每人拥有完整七维面板：基础人设 / 专业栈 / 性格底色(MBTI) / **职业信条** / **软肋盲区** / **情绪触发词** / 代表作。信条产生立场冲突，软肋让NPC会犯错，触发词让讨论有爆发力——没有人是完美的，也没有人是AI味十足的。

### 📦 拿走就能干的产品草稿包

输出不是方向建议，是8大板块的**可执行方案**：文案可录可发（不是大纲）、分镜可执行（有景别/角度/参数）、排期有里程碑、风控有具体预案。8项结构化自检确保质量——旁白字数匹配时长、标题经把关人审核、分镜总时长与规格一致。

### 🔍 联网搜索，接入真实世界

热点发现搜索微博/抖音热搜，运营讨论校准最新平台算法，竞品分析搜索同类内容表现，AIGC工具推荐验证价格和能力，突发事件卡牌优先改编真实案例——方案不活在真空里。

### 🔇 隐身记录员，不抢戏但靠谱

2位首席记录员（方哲/孟姐）不占参会名额、不参与讨论，但每轮具名小结、异常时干预、纪要末尾签名——确保决策不遗漏、讨论不跑偏、输出有追溯。

## 📖 这是什么

你扮演总策划 / 会议发起人，Skill 帮你组建一支虚拟编辑部团队来头脑风暴。

🎬 策划提创意 · 🎙️ 记者讲素材 · ✂️ 后期泼冷水 · 📱 运营追流量 · 🛡️ 把关人盯红线

就像真实的编前会。讨论结束后，你拿到的是一份**产品草稿包**：文案能直接用，分镜能直接拍，排期能直接执行。

> 💡 **核心理念：拿走就能干。**

## 🚀 快速开始

直接告诉我想做什么选题，一句话就够：

```
📝 "我想做一个关于老字号品牌焕新的短视频"
📱 "最近有个突发事件，想做一周年纪念H5"
🎥 "留守儿童专题片，不知道怎么切入"
```

只有模糊想法？说"做个爆款"或"我不知道做什么"——编辑部会联网搜索近期热点帮你拆解方向。

## 📥 安装

### Claude (Desktop / Code)

将 newmedia-jam 文件夹复制到 Claude 的 skills 目录：

```
# macOS / Linux
cp -r newmedia-jam ~/.claude/skills/newmedia-jam

# Windows (PowerShell)
Copy-Item -Recurse newmedia-jam $env:USERPROFILE\.claude\skills\newmedia-jam
```

### OpenAI Codex

```
# macOS / Linux
ln -s $(pwd)/newmedia-jam ~/.agents/skills/newmedia-jam

# Windows (PowerShell, 管理员)
New-Item -ItemType SymbolicLink -Path "$env:USERPROFILE\.agents\skills\newmedia-jam" -Target "$PWD\newmedia-jam"
```

### 项目级安装（跟随 Git 仓库）

```
cp -r newmedia-jam ./.claude/skills/newmedia-jam
```

> 📌 安装后重启 Claude 或 Codex 即可生效。更多平台安装说明见 [COMPATIBILITY.md](COMPATIBILITY.md)。

### 可选：与 PKB Framework 联动

如果你同时使用 [PKB Framework](https://github.com/MoguTown/pkb-skill)（个人知识库管理 skill），Jam 会自动检测并联动——方案自动存入知识库输出目录，策划偏好跨会话复用，讨论中可检索过往方案。详见 [PKB-FRAMEWORK-GUIDE.md](PKB-FRAMEWORK-GUIDE.md)。**不装 PKB 也不影响 Jam 全部功能。**

## 🔄 6步流程

| 步骤 | 做什么 | 你的参与 |
|------|--------|---------|
| 1️⃣ 选题拆解 | 联网搜索热点 → 5W1H+SWOT分析 → 精准追问 | 提供选题，回答3-4个问题 |
| 2️⃣ 选角组队 | 智能推荐岗位+NPC → 张力审计 | 可换人/加人/减人/自定义 |

| 4️⃣ 阶段收网 | 输出会议纪要（共识+分歧+决策追踪） | 对分歧拍板 |
| 5️⃣ 动态干预 | 换人 / 突发事件卡牌 / 继续深挖 | 灵活操作 |
| 6️⃣ 草稿包输出 | 8大板块完整方案 + 质量自检 | 审阅终稿 |

## 🎭 NPC 角色体系

8个参会岗位 31位预设角色 + 1个隐身岗位 2位首席记录员。每位角色拥有完整**七维面板**——不只是一个名字和标签，是一个有立场、有盲区、会炸毛的虚拟同事。

| 岗位 | 人数 | 风格覆盖 | 代表信条 |
|------|------|---------|---------|
| 策划/编导 | 7 | 主旋律 / 爆款 / 纪录片 / 共情 / 数据 / 互动 / 国际传播 | "没有站位的创意是空中楼阁" —老陈 |
| 记者/采编 | 5 | 暗访 / 突发 / 深度专访 / 街采vlog / 基层一线 | "信源不交叉不采用" —林晓 |
| 视觉/交互 | 4 | UI体验 / 动效 / AIGC视觉 / 插画条漫 | "视觉叙事不是装饰，是内容本身" —栗子 |
| 剪辑/后期 | 4 | 快剪 / 纪录片长线 / 特效 / AIGC辅助 | "3秒留不住人，后面白搭" —小马 |
| 音频/播客 | 1 | 播客叙事 / 声音设计 | "好声音能让闭眼的人看见画面" —声叔 |
| 运营/宣发 | 6 | 抖音 / B站 / 小红书 / 视频号 / 全平台 / 私域社群 | "内容好不好数据说了算" —罗锤 |
| 把关 | 4 | 政治导向 / 版权合规 / 舆情风控 / 内容标准 | "政治导向一票否决" —严导 |
| 记录员 🔇 | 2 | 决策精炼 / 流程守护（隐身岗位） | "讨论可以发散，纪要必须收敛" —方哲 |

> 📌 预设角色池无法覆盖时，可按描述即时生成**临时NPC**——同样遵循七维面板和发言质量要求。

## ❓ 常见问题

**🤔 我只有一个模糊的想法可以吗？**

可以。越模糊越需要编辑部帮你拆解。Skill 会联网搜索近期热点帮你找方向，哪怕只有一个关键词也能起步。

**🔄 讨论的人我可以换吗？**

随时换。对某个人不爽就说"换掉他"，想加人说"加一个懂数据可视化的人"，Skill 会按你的描述即时生成临时角色。

**📋 最后给我的东西能用吗？**

文案可直接使用（最多微调），分镜可直接拍摄（标了景别和参数），排期可直接执行（有里程碑和关键路径）。输出前自动做12项质量自检，不达标会标注。

**🔥 讨论真的会有冲突吗？**

会。冲突不是装饰，是机制保障——每轮至少1次碰撞，运营要流量把关人会亮牌，策划提大制作后期会泼冷水。没人附和没人客气。

**⏭️ 我可以跳过某些步骤吗？**

可以。想直接组团队或直接出方案，告诉我你想到哪一步。跳步输出的草稿包会标注"信息不完整，建议补充XX"。

**🤷 我不懂专业术语怎么办？**

讨论中出现的术语会自然解释。某个词不理解，直接问"XX是什么意思"。NPC会用你能懂的方式讲清楚。

**🔗 这个 skill 能和 PKB Framework 一起用吗？**

可以，且会自动联动。Jam 自动检测 PKB 环境，检测到后：方案自动存入 PKB 输出目录、读取你的策划偏好让提问更精准、检索过往方案避免重复造轮子。详见 [PKB-FRAMEWORK-GUIDE.md](PKB-FRAMEWORK-GUIDE.md)。不装 PKB 也不影响 Jam 全部功能。

**💻 在 Codex / Copilot 上能用吗？**

可以。详见 [COMPATIBILITY.md](COMPATIBILITY.md)——包含各平台差异、降级策略和移植检查清单。
