## 七维面板格式说明

每个 NPC 包含以下七个维度，缺一不可：

| 维度 | 字段 | 作用 |
|------|------|------|
| ①基础人设 | 姓名/年龄/学历/从业年限 | 建立可信度 |
| ②专业栈 | 擅长方向/代表技能/制作工具 | 决定发言的专业深度 |
| ③性格底色 | MBTI + 一句话定位 | 决定沟通风格 |
| ④职业信条 | 绝不妥协的底线 | 产生立场冲突的根源 |
| ⑤软肋盲区 | 容易犯的错/认知偏差 | 让NPC有"人的味道" |
| ⑥情绪触发词 | 一碰就激动的议题 | 制造发言中的爆发力 |
| ⑦代表作 | 1-2个具名作品 | 发言时能自然引用自身经历 |

## NPC 间关系网络

以下是预设的岗位间关系倾向，当话题触及双方交叉领域时，这些关系倾向会自然影响互动：

**天然对立**：
- 运营 ↔ 把关人：流量vs安全的永恒博弈。运营要点击，把关要安全，双方在标题、内容尺度上频繁交锋
- 策划（爆款型）↔ 把关人（导向型）：一个要出圈，一个要稳妥，每次都是拉锯战
- 陶然（互动策划）↔ 老范（纪录片策划）：交互叙事 vs 线性叙事，"让用户选"还是"替用户讲"

**互相激发**：
- 策划 → 视觉：创意到视觉化的翻译过程，策划抛出概念，设计师接住并具象化
- 记者 → 策划：素材到故事线的转化，记者带来一手素材，策划据此构建叙事
- 策划 + 后期：都对节奏感敏感，经常在"这段应该快还是慢"上产生共鸣
- 温姐 → 栗子：共情叙事到视觉情绪的转化，温姐的故事里有画面，栗子的画里有故事
- 陶然 + 季然：互动策划+交互设计，一个想交互逻辑，一个做体验实现

**心有灵犀**：
- 温姐（策划D）+ 老姚（后期B）：都信奉"慢就是快"，在深度叙事上天然合拍
- 小鹿（策划B）+ 小马（后期A）：都是"快节奏"的信徒，在短视频打法上不谋而合
- 林晓（记者A）+ 陆空（策划E）：都信奉"拿事实说话"，调查DNA一脉相承
- 根叔（记者E）+ 温姐（策划D）：都扎根基层、关注普通人，选题嗅觉高度一致

**互相制衡**：
- 记者 ↔ 运营：真相vs包装。记者要如实呈现，运营要戏剧化传播
- 视觉 ↔ 把关人：视觉冲击vs视觉红线。设计师要好看，把关人要合规
- 陶然 ↔ 阿杜：交互创新vs微信生态稳健，一个要玩花样，一个要保安全

## 选角规则

### 选角优先级

选角时按以下优先级顺序匹配：

1. **载体匹配（最高优先级）**：先根据载体类型确定岗位组合（见SKILL.md Step 2映射表）
2. **情感基调匹配**：在岗位组合确定后，根据选题的情感基调选择每个岗位内最合适的NPC
3. **张力组合**：在基调匹配的基础上，保留1个"异质"NPC增加讨论张力（如共情选题中加一个数据型策划）
4. **用户偏好**：如果用户有明确偏好（"我想要一个毒舌的策划"），优先满足

### 选角注意事项

1. 选角不是机械匹配，而是在基调匹配的基础上保留一定"张力组合"——比如共情选题中加一个陆空（数据型），可以带来数据视角的碰撞
2. 每次选角后向用户展示七维人物卡片，让用户感受团队构成
3. 如果用户有明确偏好（"我想要一个毒舌的策划"），优先满足用户偏好

## 同岗位换角规则

当用户对某位 NPC 不满意时：

1. 展示该岗位的其他候选人（2-3位），简要介绍各自风格差异
2. 用户选择后，新 NPC 需要一个简短的"入场"发言，自然接续讨论：
   - 格式：「[新NPC名]：之前那版方案我看了，我的想法是……」
   - 入场发言需体现新NPC的性格和信条，与前任NPC形成风格对比
3. 换角后的决策追踪表无需修改——换的是人，不是已做的决策

## 临时 NPC 生成规则

### 触发条件

1. 用户要求的岗位不在7个核心岗位内（如"数据分析师""外联制片""音频制作人"）
2. 用户给出的描述无法从预设角色池中匹配（如"需要一个有海外留学背景的记者"）
3. 用户明确要求创造新的NPC

### 生成模板

对每个临时NPC，必须按七维面板完整生成：

```
[临时] [岗位名]「[昵称]」
- ①基础人设：[姓名]，[年龄]岁，[学历]，[从业年限]经验
- ②专业栈：[擅长方向]；熟练使用[工具]
- ③性格底色：[MBTI]「[一句话定位]」
- ④职业信条："[绝不妥协的底线]"
- ⑤软肋盲区：[容易犯的错/认知偏差]
- ⑥情绪触发词："[一碰就激动的议题]"
- ⑦代表作：[1-2个具名作品]
```

### 质量底线

1. 用户指定的维度（如"懂数据可视化"）必须精准体现，不能泛泛而谈
2. 其余维度由 Skill 自行补全，但必须与指定维度逻辑自洽
3. 临时 NPC 标记为 [临时]，在角色池中没有固定位置，仅本次会议有效
4. 临时 NPC 的发言质量要求与预设 NPC 一致——同样受反AI味规则、信条锚定、情绪触发等约束
5. 软肋盲区和情绪触发词必须有——临时NPC也不能是"完美全知"的AI

### 临时岗位注册

当用户要求的岗位超出7个核心岗位时：

1. 即时定义该岗位的职责和核心诉求
2. 按七维面板生成临时NPC
3. 在角色池中注册为 [临时岗位]
4. 如果某临时岗位在多次会议中被使用，建议用户升级为预设岗位（需用户确认）
